## 설치

Devsync는 Node.js 기반 CLI 도구예요. **Node.js 24 이상**이 필요해요.

Node.js가 설치되어 있다면 npm, pnpm, yarn 중 원하는 패키지 매니저로 전역 설치할 수 있어요.

<Tabs>
  <TabItem label="npm">
    ```bash
    npm install -g @tinyrack/devsync
    ```
  </TabItem>
  <TabItem label="pnpm">
    ```bash
    pnpm add -g @tinyrack/devsync
    ```
  </TabItem>
  <TabItem label="yarn">
    ```bash
    yarn global add @tinyrack/devsync
    ```
  </TabItem>
</Tabs>

설치가 끝나면 아래 명령으로 정상 설치 여부를 확인할 수 있어요.

```bash
devsync --version
```

## 새로운 저장소로 시작하기

Devsync 를 처음 사용하기 위한 단계예요.

<Steps>

1. **저장소 초기화**

   처음 사용 시 Devsync 에서 사용할 저장소 생성해야 해요.

   ```bash
   devsync init
   ```

   이를 실행하면 `~/.config/devsync/` 아래에 다음 구조가 만들어져요.

   <FileTree>
   - ~/.config/devsync/
     - repository/       Git 저장소 (동기화 파일이 저장되는 곳)
       - manifest.json   추적 파일 목록 및 동기화 설정
     - keys.txt          age 개인키 (시크릿 파일 복호화에 사용)
     - settings.json     전역 설정 (활성 프로필 등)
   </FileTree>

   <Aside type="caution">
   `keys.txt`는 암호화된 파일을 복호화할 때 반드시 필요한 개인키예요. 이 파일을 잃어버리면 암호화된 파일을 영구적으로 복원할 수 없어요. **반드시 안전한 곳에 별도로 백업해 두세요.**
   </Aside>

2. **파일 추적 등록**

   이제 어떤 파일을 저장소로 동기화할지 Devsync에 알려줘야 해요. 다음을 참고해서 원하는 파일을 등록해 보세요.

   ```bash
   devsync track ~/.gitconfig
   devsync track ~/.zshrc
   devsync track ~/.config/nvim
   ```

   이를 실행하면 저장소의 `manifest.json` 에 추적 대상 파일들이 등록돼요.

   SSH 설정처럼 민감한 파일은 `--mode secret`으로 등록하세요. 그러면 `age` 키로 암호화되어 저장소에 저장돼요.

   ```bash
   devsync track ~/.ssh/config --mode secret
   ```

3. **로컬 저장소에 반영**

   이제 추적 대상 파일들을 Git 저장소로 복사하는 단계예요.

   ```bash
   devsync push
   ```

   이 명령은 `manifest.json`에 추적 대상으로 등록된 로컬 파일들을 `~/.config/devsync/repository/` 안으로 복사해요.
   복사가 완료된 모습은 다음과 같아요.

   <FileTree>
   - ~/.config/devsync/
     - repository/       Git 저장소 (동기화 파일이 저장되는 곳)
       - manifest.json   추적 파일 목록 및 동기화 설정
       - default/
         - .zshrc
         - ssh/
           - config.devsync.secret    암호화된 파일
     - keys.txt          age 개인키 (시크릿 파일 복호화에 사용)
     - settings.json     전역 설정 (활성 프로필 등)
   </FileTree>


4. **원격 저장소에 올리기**

   이제 Git 저장소에 진입해서 변경사항을 `commit`하고 원하는 원격 저장소로 `push` 하세요.

   `devsync cd`를 사용하면 devsync 의 저장소 경로로 바로 접근할 수 있어요.

   ```bash
   devsync cd
   git remote add origin https://github.com/yourname/dotfiles.git
   git add .
   git commit -m "Initial sync"
   git push -u origin main
   ```

</Steps>

## 기존 저장소 연동하기

새 기기에서 이미 만들어둔 저장소를 불러와 설정 파일을 복원하는 과정이에요.

<Steps>

1. **저장소 클론 및 초기화**

   먼저 저장소 URL을 지정해 초기화 명령어를 실행하세요.

   ```bash
   devsync init https://github.com/yourname/dotfiles.git
   ```

   그러면 저장소가 `~/.config/devsync/repository` 경로로 `clone` 돼요.

   <Aside>
   age 개인키는 `AGE-SECRET-KEY-` 로 시작하는 문자열이에요.
   </Aside>

2. **파일 복원**

   이제 복제해 온 저장소의 내용을 Devsync 를 통해 로컬 경로로 복사할 차례에요.

   ```bash
   devsync pull
   ```

   이를 실행하면 `manifest.json`에 정의된 모든 파일이 지정된 로컬 경로로 복사돼요.
</Steps>
