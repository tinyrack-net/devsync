이 가이드에서는 devsync 사용의 전체 생명주기를 다뤄요 — 첫 기기 설정부터
여러 기기를 매일 동기화하는 것까지요.

## 기기 A에서의 초기 설정

<Steps>

1. **devsync 초기화**

   ```bash
   devsync init ~/dotfiles
   ```

   `~/dotfiles`에 Git으로 관리되는 sync 디렉터리가 생성돼요.

2. **파일 추적**

   ```bash
   devsync track ~/.gitconfig
   devsync track ~/.config/starship.toml
   devsync track ~/.ssh/config --secret
   ```

   각 명령어는 sync 디렉터리 안의 `manifest.json`에 항목을 추가해요.

3. **로컬 파일을 sync 디렉터리로 push**

   ```bash
   devsync push
   ```

4. **Git으로 commit & 리모트에 push**

   devsync는 Git commit을 자동으로 만들지 않아요 — 모든 통제권은 사용자에게 있어요:

   ```bash
   devsync cd   # sync 디렉터리 안에서 셸을 열어요
   git add -A && git commit -m "initial sync"
   git remote add origin git@github.com:you/dotfiles.git
   git push -u origin main
   exit
   ```

</Steps>

<Aside type="tip">
push나 pull 전에 `devsync status`를 실행하면 어떤 변경이 있을지 미리 확인할 수
있어요.
</Aside>

## 기기 B 설정

<Steps>

1. **리모트 URL로 초기화**

   ```bash
   devsync init ~/dotfiles --url git@github.com:you/dotfiles.git
   ```

2. **파일을 로컬 경로로 pull**

   ```bash
   devsync pull
   ```

   devsync가 `manifest.json`을 읽고, 플랫폼별 경로를 해석하고, 시크릿을
   복호화한 뒤 모든 파일을 올바른 위치에 써줘요.

</Steps>

## 일상적인 워크플로우

두 기기 모두 설정을 마치면, 매일의 흐름은 간단하고 예측 가능해요.

**변경사항이 있는 기기에서:**

```bash
devsync push
devsync cd
git add -A && git commit -m "update shell aliases"
git push
exit
```

**다른 기기에서:**

```bash
devsync cd
git pull
exit
devsync pull
```

<Aside>
패턴은 항상 한쪽에서 **devsync push → git push**, 다른 쪽에서
**git pull → devsync pull**이에요.
</Aside>

## 충돌 처리

sync 디렉터리는 일반 Git 저장소이므로 충돌도 일반적인 방법으로 해결해요:

```bash
devsync cd
git pull            # merge 충돌이 발생할 수 있어요
# 해당 파일에서 충돌을 해결
git add -A && git commit -m "resolve merge conflict"
git push
exit
devsync pull        # 해결된 상태를 로컬에 적용
```

<Aside type="caution">
두 기기가 같은 파일에 다른 수정을 push한 뒤 어느 쪽도 `git pull`을 하지 않으면,
일반적인 Git merge 충돌이 발생해요. sync 디렉터리 안에서 직접 해결하세요 —
devsync 자체는 자동 병합을 하지 않아요.
</Aside>

## 상태 확인

```bash
devsync doctor
```

sync 디렉터리의 존재 여부, Git 설정, 암호화 키, 추적 경로의 접근 가능 여부를
확인해 줘요.

## 기기 추가하기

세 번째(또는 네 번째, 다섯 번째) 기기를 추가하는 것도 기기 B와 같은 과정이에요
— 리모트 URL로 `devsync init`을 하고 `devsync pull`을 하면 돼요. 이후의 일상적인
push/pull 사이클은 모든 기기에서 동일해요.

<Aside type="tip">
새 기기가 다른 OS라도 devsync가 플랫폼별 경로를 자동으로 해석해 줘요.
[플랫폼별 경로](/ko/guides/platform-specific-paths/) 가이드를 참고하세요.
</Aside>

## 빠른 참조

| 작업                        | 명령어                                                    |
| --------------------------- | --------------------------------------------------------- |
| 변경사항 미리보기           | `devsync status`                                          |
| 로컬 수정을 업스트림으로    | `devsync push` → `git add -A && git commit` → `git push` |
| 업스트림 수정 가져오기      | `git pull` → `devsync pull`                               |
| 충돌 해결                   | `devsync cd` → 파일 수정 → `git commit` → `git push`     |
| 설정 상태 확인              | `devsync doctor`                                          |
