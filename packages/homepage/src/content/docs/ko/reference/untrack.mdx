## 형식

```bash
devsync untrack <path>
```

## 설명

`devsync untrack`은 경로를 `manifest.json`에서 제거해서 이후 `push`와 `pull` 작업에서 제외해요.

로컬 절대 경로(예: `~/.gitconfig`)와 저장소 기준 상대 경로(예: `.gitconfig`) 모두 사용할 수 있어요. 두 형식 모두 같은 manifest 항목을 가리켜요.

경로를 언트랙해도 **로컬 디스크의 파일은 삭제되지 않아요**. 동기화 저장소의 기존 파일도 제거되지 않아요. 변경 사항은 다음 `push` 또는 `pull` 실행 시부터 반영돼요.

## 인수

| 인수 | 설명 |
|------|------|
| `path` | 추적을 중단할 로컬 절대 경로 또는 저장소 기준 상대 경로. |

## 옵션

| 플래그 | 설명 |
|--------|------|
| `--verbose` | 상세 출력을 표시해요. |

## 예시

```bash
# 로컬 경로로 언트랙
devsync untrack ~/.gitconfig

# 틸다 경로로 언트랙
devsync untrack ~/.config/nvim

# 저장소 기준 상대 경로로 언트랙
devsync untrack .gitconfig
```

## 참고

<Aside type="note">
  언트랙 후에도 동기화 저장소(`~/.config/devsync/repository/`)에는 기존 파일이 남아 있어요. 완전히 제거하려면 [`devsync cd`](/ko/reference/cd)로 저장소에 진입한 뒤 파일을 직접 삭제하고 커밋하면 돼요.
</Aside>

<Aside type="caution">
  디렉터리를 언트랙해도 그 안에서 별도로 등록된 하위 경로 항목은 자동으로 제거되지 않아요. 각 하위 경로에 대해 개별적으로 `untrack`을 실행하거나, `manifest.json`을 직접 편집해서 정리해야 해요.
</Aside>
