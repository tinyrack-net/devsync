## 형식

```bash
devsync push
```

## 설명

`devsync push`는 추적 중인 모든 로컬 파일의 현재 상태를 읽어 동기화 저장소(`~/.config/devsync/repository/`)를 로컬과 일치하도록 업데이트해요. **로컬 파일이 항상 진실의 원천**이에요.

`mode: secret`으로 등록된 항목은 저장소에 쓰기 전에 age로 암호화돼요. 파일은 `.devsync.secret` 확장자로 저장되어 평문으로 커밋되는 일이 없어요.

push 후에는 동기화 저장소에 최신 설정 스냅샷이 담겨요. [`devsync cd`](/ko/reference/cd)로 저장소에 진입해서 `git commit`과 `git push`로 원격에 보낼 수 있어요.

`push`는 git 커밋을 자동으로 생성하지 않아요.

## 옵션

| 플래그 | 설명 |
|--------|------|
| `--dry-run` | 실제로 쓰지 않고 변경될 내용만 미리 보여요. |
| `--profile NAME` | 이번 실행에 한해 활성 프로파일을 재정의해요. `settings.json`에는 저장되지 않아요. |
| `--verbose` | 작업 중 항목별·파일별 상세 내용을 표시해요. |

## 일반적인 워크플로우

```bash
# 1. 평소처럼 설정 파일 편집
vim ~/.zshrc

# 2. 변경된 내용 확인
devsync status

# 3. 로컬 변경 사항을 동기화 저장소에 반영
devsync push

# 4. git으로 원격에 푸시
devsync cd
# (이제 ~/.config/devsync/repository/ 안에 있어요)
git add -A
git commit -m "update zshrc"
git push
exit
```

## 참고

<Aside type="note">
  `push`는 원격 서버와 통신하지 않아요. 오직 로컬 동기화 저장소 디렉터리에만 써요. 원격 git 호스트에 동기화하려면 git 명령으로 별도로 진행해야 해요.
</Aside>

<Aside type="caution">
  `mode: ignore`인 항목은 push 시에도 완전히 건너뛰어요. 두 위치에 파일이 있어도 읽거나 쓰지 않아요.
</Aside>
