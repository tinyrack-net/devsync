로컬 파일이나 디렉터리를 동기화 대상으로 등록해요.

홈 디렉터리 안에 있는 파일이나 디렉터리를 하나 이상 등록하면, dotweave가 sync 디렉터리에 미러링해요. 이미 추적 중인 대상은 mode만 갱신돼요. 추적 중인 디렉터리 안의 하위 경로를 지정해서 특정 mode를 가진 자식 항목을 만들 수도 있어요.

## Usage

```bash
dotweave track <targets...> [flags]
```

## Arguments

| Argument       | Description                                            |
| -------------- | ------------------------------------------------------ |
| `<targets...>` | 추적할 로컬 경로 또는 저장소 경로를 하나 이상 지정해요.  |

## Flags

| Flag                  | Description                                                   |
| --------------------- | ------------------------------------------------------------- |
| `--mode <mode>`       | sync mode: `normal`(기본값), `secret`, `ignore` 중 하나예요.         |
| `--profiles <names>`  | 특정 profile에서만 동기화하도록 제한해요(쉼표로 구분).      |
| `--repoPath <path>`   | profile 디렉터리 아래의 저장소 상대 경로예요.         |
| `--verbose`           | 상세한 출력을 보여줘요.                                         |

## Examples

설정 파일 하나를 추적해요:

```bash
dotweave track ~/.config/starship.toml
```

secret이 포함된 디렉터리를 추적해요:

```bash
dotweave track ~/.ssh --mode secret
```

특정 profile에만 파일을 추적해요:

```bash
dotweave track ~/.config/work-proxy.conf --profiles work
```

추적 중인 디렉터리 안의 자식 경로에 mode를 재정의해요:

```bash
dotweave track .ssh/config --mode ignore
```

<Aside>
`track`은 `manifest.jsonc`만 갱신해요. 파일을 sync 디렉터리에 복사하려면 `dotweave push`를 실행하세요.
</Aside>
