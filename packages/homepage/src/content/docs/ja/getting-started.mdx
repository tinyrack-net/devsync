## 前提条件

- **Node.js 24**以降

## インストール

```bash
npm install -g @tinyrack/dotweave
```

---

## 新規セットアップ

dotweaveを初めて使う場合は、以下の手順で新しいsyncディレクトリを作成します。

<Steps>

1. **初期化**

   シークレット暗号化用のageキーを含む新しいsyncディレクトリを作成します。

   ```bash
   dotweave init
   ```

   これにより、`~/.config/dotweave/`にGitリポジトリとageのidentityが設定されます。

2. **ファイルをトラッキング**

   同期したいファイルをdotweaveに登録します。例えば：

   ```bash
   dotweave track ~/.zshrc
   dotweave track ~/.config/starship.toml
   dotweave track ~/.config/nvim
   ```

   SSHキーなどの機密ファイルにはsecretモードを使います：

   ```bash
   dotweave track --mode secret ~/.ssh/config
   ```

3. **syncディレクトリにpush**

   トラッキング中のファイルをsyncディレクトリにコピーします：

   ```bash
   dotweave push
   ```

4. **Gitでcommit & push**

   syncディレクトリを開いてリモートにpushします：

   ```bash
   dotweave cd
   git add -A && git commit -m "Initial sync"
   git remote add origin <your-repo-url>
   git push -u origin main
   ```

</Steps>

<Aside type="tip">
  セットアップが完了したら、[ディレクトリ構造](/ja/guides/directory-structure/)ガイドで、dotweaveが設定ファイルと同期データをどこに保存するかを確認してみてください。
</Aside>

## 既存のsyncディレクトリに接続する

新しいマシンで、既存のsyncディレクトリをcloneします：

<Steps>

1. **cloneして初期化**

   ```bash
   dotweave init <your-repo-url>
   ```

   リポジトリが`~/.config/dotweave/repository/`にcloneされ、新しいageのidentityが設定されます。別のマシンのageプライベートキーをすでにお持ちの場合は、`--key`で指定できます：

   ```bash
   dotweave init <your-repo-url> --key "AGE-SECRET-KEY-1..."
   ```

2. **ファイルをpull**

   トラッキング中のすべてのファイルをローカルパスに復元します：

   ```bash
   dotweave pull
   ```

</Steps>

<Aside type="note">
  syncディレクトリに暗号化されたシークレットが含まれている場合、すべてのマシンで同じageプライベートキーが必要です。詳しくは[シークレットファイルの同期](/ja/guides/syncing-secrets/)をご覧ください。
</Aside>
