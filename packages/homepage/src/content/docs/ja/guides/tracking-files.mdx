dotweaveで何かを同期するには、まず**トラッキング**する必要があります。トラッキングとは、「このファイルを管理したい」とdotweaveに伝えることで、`manifest.jsonc`にエントリが追加されます。ファイルのコピーやpushは、`dotweave push`を実行するまで行われません。

---

## ファイルをトラッキングする

`dotweave track`にファイルパスを渡します：

```bash
dotweave track ~/.gitconfig
dotweave track ~/.zshrc
```

これにより`manifest.jsonc`にエントリが追加されるだけです。ファイル自体はそのまま残ります。実際にsyncディレクトリにコピーするには、別途`dotweave push`を実行してください。

---

## ディレクトリをトラッキングする

ディレクトリ全体をトラッキングすることもできます。pushやpull時に、その中のすべてのファイルが再帰的に含まれます。

```bash
dotweave track ~/.config/nvim
dotweave track ~/.config/fish
```

<Aside type="tip">
ディレクトリをトラッキングすると、後から追加されたファイルも次回のpush時に自動的に含まれます — 個別にトラッキングする必要はありません。
</Aside>

---

## 同期モードを設定する

デフォルトでは、トラッキングされたエントリは`normal`モードです。`--mode`で変更できます：

```bash
dotweave track ~/.ssh/config --mode secret
dotweave track ~/.local/share/cache-stuff --mode ignore
```

各モードの詳細は[同期モード](/ja/guides/sync-modes)をご覧ください。

---

## 特定のプロファイルに限定する

`--profiles`を使って、エントリを1つまたは複数のプロファイルに割り当てます。アクティブなプロファイルが一致するマシンだけがそのエントリを同期します。

```bash
dotweave track ~/.config/work-tool --profiles work
dotweave track ~/.config/shared-tool --profiles work,personal
```

`--profiles`を付けないエントリは`default`プロファイルに入り、すべてのマシンで同期されます。詳しくは[プロファイル](/ja/guides/profiles)をご覧ください。

---

## リポジトリパスのカスタマイズ

デフォルトでは、dotweaveはホームディレクトリからの相対パスをsyncディレクトリ内にミラーリングします。リポジトリ内で別のパスを使いたい場合は、`--repoPath`を指定します：

```bash
dotweave track ~/Projects/scripts/deploy.sh --repoPath scripts/deploy.sh
```

---

## ユースケース

**ホワイトリスト方式** — 同期したい特定のファイルだけをトラッキング：

```bash
dotweave track ~/.gitconfig
dotweave track ~/.zshrc
dotweave track ~/.config/starship.toml
dotweave push
```

**ブラックリスト方式** — ディレクトリ全体をトラッキングし、特定のファイルをignore：

```bash
dotweave track ~/.config/nvim
dotweave track ~/.config/nvim/plugin/packer_compiled.lua --mode ignore
dotweave push
```

---

## トラッキング解除

気が変わった場合は、`dotweave untrack`でエントリを削除できます：

```bash
dotweave untrack ~/.gitconfig
```

これは`manifest.jsonc`からエントリを削除するだけです。ローカルファイルやsyncディレクトリ内の既存のコピーは**削除されません**。

---

## トラッキング状況の確認

`dotweave status`で、すべてのトラッキングエントリと現在の状態を確認できます：

```bash
dotweave status
```

`--dry-run`を付けると、変更を適用せずにプレビューできます：

```bash
dotweave push --dry-run
dotweave pull --dry-run
```
