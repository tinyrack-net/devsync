description: ファイルやディレクトリを同期対象として登録する方法。
---

devsyncで何かを同期するには、まず**トラッキング**する必要があります。トラッキングとは、「このファイルを管理したい」とdevsyncに伝えることで、`manifest.json`にエントリが追加されます。ファイルのコピーやpushは、`devsync push`を実行するまで行われません。

---

## ファイルをトラッキングする

`devsync track`にファイルパスを渡します：

```bash
devsync track ~/.gitconfig
devsync track ~/.zshrc
```

これにより`manifest.json`にエントリが追加されるだけです。ファイル自体はそのまま残ります。実際にsyncディレクトリにコピーするには、別途`devsync push`を実行してください。

---

## ディレクトリをトラッキングする

ディレクトリ全体をトラッキングすることもできます。pushやpull時に、その中のすべてのファイルが再帰的に含まれます。

```bash
devsync track ~/.config/nvim
devsync track ~/.config/fish
```

<Aside type="tip">
ディレクトリをトラッキングすると、後から追加されたファイルも次回のpush時に自動的に含まれます — 個別にトラッキングする必要はありません。
</Aside>

---

## 同期モードを設定する

デフォルトでは、トラッキングされたエントリは`normal`モードです。`--mode`で変更できます：

```bash
devsync track ~/.ssh/config --mode secret
devsync track ~/.local/share/cache-stuff --mode ignore
```

各モードの詳細は[同期モード](/ja/guides/sync-modes)をご覧ください。

---

## 特定のプロファイルに限定する

`--profiles`を使って、エントリを1つまたは複数のプロファイルに割り当てます。アクティブなプロファイルが一致するマシンだけがそのエントリを同期します。

```bash
devsync track ~/.config/work-tool --profiles work
devsync track ~/.config/shared-tool --profiles work,personal
```

`--profiles`を付けないエントリは`default`プロファイルに入り、すべてのマシンで同期されます。詳しくは[プロファイル](/ja/guides/profiles)をご覧ください。

---

## リポジトリパスのカスタマイズ

デフォルトでは、devsyncはホームディレクトリからの相対パスをsyncディレクトリ内にミラーリングします。リポジトリ内で別のパスを使いたい場合は、`--repoPath`を指定します：

```bash
devsync track ~/Projects/scripts/deploy.sh --repoPath scripts/deploy.sh
```

---

## ユースケース

**ホワイトリスト方式** — 同期したい特定のファイルだけをトラッキング：

```bash
devsync track ~/.gitconfig
devsync track ~/.zshrc
devsync track ~/.config/starship.toml
devsync push
```

**ブラックリスト方式** — ディレクトリ全体をトラッキングし、特定のファイルをignore：

```bash
devsync track ~/.config/nvim
devsync track ~/.config/nvim/plugin/packer_compiled.lua --mode ignore
devsync push
```

---

## トラッキング解除

気が変わった場合は、`devsync untrack`でエントリを削除できます：

```bash
devsync untrack ~/.gitconfig
```

これは`manifest.json`からエントリを削除するだけです。ローカルファイルやsyncディレクトリ内の既存のコピーは**削除されません**。

---

## トラッキング状況の確認

`devsync status`で、すべてのトラッキングエントリと現在の状態を確認できます：

```bash
devsync status
```

`--dry-run`を付けると、変更を適用せずにプレビューできます：

```bash
devsync push --dry-run
devsync pull --dry-run
```
