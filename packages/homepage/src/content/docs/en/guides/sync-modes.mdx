Every tracked entry has a **sync mode** that controls how devsync handles it during push and pull. There are three modes to choose from: `normal`, `secret`, and `ignore`.

---

## At a Glance

| Mode | Stored in sync directory | Encrypted | Synced on push/pull |
|---|---|---|---|
| `normal` | Yes | No | Yes |
| `secret` | Yes (`.age` suffix) | Yes (age) | Yes |
| `ignore` | No | — | No |

---

## normal (default)

Files are copied to the sync directory exactly as-is. No transformation, no encryption — what you see is what you get.

```bash
devsync track ~/.gitconfig
devsync track ~/.config/starship.toml
devsync track ~/.config/nvim
```

**Great for:** shell configs (`.zshrc`, `.bashrc`), editor settings, Git config, tool configs like `.tmux.conf` or `starship.toml` — basically anything that doesn't contain secrets.

---

## secret

Files are encrypted with [age](https://age-encryption.org/) before being stored in the sync directory. The stored file gets an `.age` suffix.

```bash
devsync track ~/.ssh/config --mode secret
devsync track ~/.config/gh/hosts.yml --mode secret
devsync track ~/.aws/credentials --mode secret
```

**Great for:** SSH keys and config, API tokens, `.env` files with real credentials, GPG keys — anything you wouldn't want exposed if your repository went public.

During `pull`, devsync decrypts the file with your age private key before writing it to disk. See [Syncing Secrets](/en/guides/syncing-secrets) for the full setup guide.

<Aside type="caution">
Secret mode encrypts the file **content**, but the filename and path are still visible in the sync directory. Keep that in mind if the path itself is sensitive.
</Aside>

---

## ignore

The entry stays in `manifest.json` but is completely skipped during push and pull. Nothing is read, written, or transferred.

```bash
devsync track ~/.config/some-tool/cache --mode ignore
devsync track ~/.local/share/generated-config --mode ignore
```

**Great for:**
- Temporarily disabling an entry without removing it from the sync config
- OS-specific files that shouldn't sync on the wrong platform
- Machine-specific configs you want to remember but not replicate

---

## Changing a Mode

Run `devsync track` again with the new `--mode` flag. The existing entry gets updated:

```bash
# Originally tracked as normal
devsync track ~/.ssh/config

# Upgrade to secret
devsync track ~/.ssh/config --mode secret
```

---

## Per-Entry Overrides

You can mix modes within a tracked directory. For example, track a directory as `normal` but override a specific child as `secret`:

```bash
devsync track ~/.config/my-tool
devsync track ~/.config/my-tool/credentials.json --mode secret
```

The directory is synced normally, but `credentials.json` inside it gets encrypted. Child entries always take priority over their parent.
