Before devsync can sync anything, you need to **track** it. Tracking tells devsync "hey, I care about this file" by adding an entry to `manifest.jsonc`. Nothing gets copied or pushed until you explicitly run `devsync push` afterward.

---

## Tracking a File

Pass any file path to `devsync track`:

```bash
devsync track ~/.gitconfig
devsync track ~/.zshrc
```

This adds entries to `manifest.jsonc` — that's it. Your files stay exactly where they are. To actually copy them into the sync directory, run `devsync push` as a separate step.

---

## Tracking a Directory

You can track an entire directory. Every file inside it (recursively) will be included when you push and pull.

```bash
devsync track ~/.config/nvim
devsync track ~/.config/fish
```

<Aside type="tip">
When you track a directory, new files added inside it later are automatically included in the next push — no need to track them individually.
</Aside>

---

## Setting a Sync Mode

By default, tracked entries use `normal` mode. You can change this with `--mode`:

```bash
devsync track ~/.ssh/config --mode secret
devsync track ~/.local/share/cache-stuff --mode ignore
```

See [Sync Modes](/en/guides/sync-modes) for a full breakdown of each mode.

---

## Limiting to Specific Profiles

Use `--profiles` to assign an entry to one or more profiles. Only machines with a matching active profile will sync that entry.

```bash
devsync track ~/.config/work-tool --profiles work
devsync track ~/.config/shared-tool --profiles work,personal
```

Entries without `--profiles` land in the `default` profile and sync everywhere. See [Profiles](/en/guides/profiles) for details.

---

## Customizing the Repository Path

By default, devsync mirrors your file's path relative to your home directory inside the sync directory. If you want a different path in the repository, use `--repoPath`:

```bash
devsync track ~/Projects/scripts/deploy.sh --repoPath scripts/deploy.sh
```

---

## Use Cases

**Whitelist approach** — track only the specific files you want synced:

```bash
devsync track ~/.gitconfig
devsync track ~/.zshrc
devsync track ~/.config/starship.toml
devsync push
```

**Blacklist approach** — track a whole directory, then ignore specific children:

```bash
devsync track ~/.config/nvim
devsync track ~/.config/nvim/plugin/packer_compiled.lua --mode ignore
devsync push
```

---

## Untracking

Changed your mind? Remove an entry with `devsync untrack`:

```bash
devsync untrack ~/.gitconfig
```

This removes the entry from `manifest.jsonc`. It does **not** delete the local file or any existing copy in the sync directory.

---

## Checking What's Tracked

Use `devsync status` to see all tracked entries and their current state:

```bash
devsync status
```

Add `--dry-run` to push or pull to preview changes without applying them:

```bash
devsync push --dry-run
devsync pull --dry-run
```
