## Synopsis

```bash
devsync pull
```

## Description

`devsync pull` reads artifacts from the sync repository (`~/.config/devsync/sync/`) and materializes them onto the corresponding local paths. It is the reverse of [`devsync push`](/en/reference/push).

For entries with `mode: secret`, the artifact is decrypted using the configured age identity before being written to disk. The decrypted content is written to the local path; the `.devsync.secret` artifact in the repository is never changed.

After writing each file, `pull` restores the original file permissions.

`pull` does **not** run `git pull`. It only reads what is already present in the local sync repository directory. To receive changes from a remote first, use `git pull` inside the sync directory (via [`devsync cd`](/en/reference/cd)), then run `devsync pull`.

## Options

| Flag | Description |
|------|-------------|
| `--dry-run` | Preview what would be written locally without making any changes. |
| `--profile NAME` | Override the active profile for this single run. Does not persist to `settings.json`. |
| `--verbose` | Show per-entry and per-file detail during the operation. |

## Examples

<Tabs>
  <TabItem label="Basic">
    ```bash
    # Apply the current sync repository state to local files
    devsync pull
    ```
  </TabItem>
  <TabItem label="Dry run">
    ```bash
    # Preview what pull would write without making changes
    devsync pull --dry-run
    ```
  </TabItem>
  <TabItem label="Profile override">
    ```bash
    # Pull only entries for the "personal" profile (one-time override)
    devsync pull --profile personal
    ```
  </TabItem>
</Tabs>

## Typical workflow on a new machine

```bash
# 1. Initialize by cloning your existing dotfiles repo
devsync init https://github.com/yourname/dotfiles.git

# 2. Preview what would be applied
devsync pull --dry-run

# 3. Apply everything
devsync pull
```

## Notes

<Aside type="note">
  `pull` does not contact any remote git server. To pull changes from a remote first, run git commands inside the sync directory and then run `devsync pull`.
</Aside>

<Aside type="caution">
  `pull` will overwrite local files with the repository versions. Run `devsync status` first to understand what will change, especially on machines where the local files may have diverged.
</Aside>

<Aside type="caution">
  Entries with `mode: ignore` are skipped entirely during pull, even if a corresponding artifact exists in the repository.
</Aside>
