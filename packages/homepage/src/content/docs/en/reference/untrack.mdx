Stop tracking a synced path.

Remove a tracked root entry or a nested override from devsync configuration. This only updates the sync config; actual file changes happen on the next `push` or `pull`. Use a local path to remove the main tracked target, or use a repository-relative child path inside a tracked directory to remove only that override.

## Usage

```bash
devsync untrack <target> [flags]
```

## Arguments

| Argument   | Description                                             |
| ---------- | ------------------------------------------------------- |
| `<target>` | Tracked local path or repository path to stop tracking. |

## Flags

| Flag        | Description            |
| ----------- | ---------------------- |
| `--verbose` | Show detailed output.  |

## Examples

Stop tracking a config file:

```bash
devsync untrack ~/.config/starship.toml
```

Remove a child override inside a tracked directory:

```bash
devsync untrack .ssh/config
```

<Aside>
`untrack` only updates `manifest.json`. It does **not** delete local files or sync directory artifacts.
</Aside>
